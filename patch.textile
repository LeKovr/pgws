h1. Инструкция по оформлению патчей

Разработчик, не имеющий доступа на запись в репозиторий, для демонстрации результатов работы по задаче, должен выполнить следующие действия:

1. Запросить обновление github (пост в задаче)
2. Обновить локальную копию до актуальной (git pull + см [[upgrade|Инструкцию по обновлению]])
3. Поместить на локальную копию сделанные изменения
4. Проверить работоспособность результата
5. Сформировать патч со сделанными изменениями (git diff > chg_BB.diff)
6. Если в этом файле - все сделанные изменения - переименовать его в pgws_AAA_chg_BB.diff и приложить к задаче в РМ
7. Если кроме патча есть еще бинарные файлы, поместить их и патч в архив pgws_AAA_chg_BB.zip и добавить в него файл README с указанием, куда помещать бинарные файлы

Обозначения:

AAA - номер задачи в РМ
BB - номер версии изменения

Например: pgws-001_chg_04
